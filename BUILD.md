# Сборка Quant Voice

## Первый запуск — одна команда

```bash
cd ~/Library/Mobile\ Documents/com~apple~CloudDocs/Claude-Projects/Quant-Voice && chmod +x scripts/build.sh && ./scripts/build.sh
```

Скрипт соберёт приложение и положит его в `~/QuantVoice-build/QuantVoice.app`.

Первая сборка идёт дольше остальных — качается WhisperKit.

## Что нужно на маке

**Xcode не нужен.** Достаточно Command Line Tools с SDK 26 или новее — сборка идёт через SwiftPM напрямую.

Проверить:

```bash
xcrun --show-sdk-version   # должно быть 26.x
swift --version            # должно быть 6.x
```

Если SDK старше 26 — обнови Command Line Tools: `xcode-select --install`. Код обращается к системному движку распознавания из macOS 26, в более старом SDK этих классов просто нет.

## Что делать после сборки

**1. Запусти приложение.**

```bash
open ~/QuantVoice-build/QuantVoice.app
```

Иконка микрофона появится в строке меню справа. В Dock ничего не появится — так и задумано.

**2. Выдай два разрешения.** Микрофон система спросит сама при первой записи. Accessibility — вручную: Системные настройки → Конфиденциальность и безопасность → Универсальный доступ → добавить QuantVoice. Без него не работает ни перехват хоткея, ни вставка текста.

**3. Открой логи** — в меню приложения пункт «Открыть логи», или сразу:

```bash
open ~/Library/Logs/QuantVoice/
```

В начале лога лежит блок «Диагностика Quant Voice»: какие движки распознавания нашлись, какие языки они знают, есть ли русский в системном движке Apple. Это ответ на главный открытый вопрос проекта. Пришли мне этот блок.

**4. Попробуй продиктовать.** Зажми ⌥Space, скажи фразу, отпусти. Или коротко нажми ⌥Space, скажи, нажми ещё раз. Esc отменяет.

Учти: если системный движок Apple не поддержит русский, приложение уйдёт на WhisperKit, а его модель (~600 МБ) ещё не скачана. Тогда распознавание не сработает и в логе будет об этом сказано. Это ожидаемо — загрузку моделей делаем на следующем шаге.

## Если сборка упала

Это нормально для первого раза: код написан без компилятора под рукой. Скопируй ошибки из вывода терминала и пришли — разберём.

Самые вероятные места перечислены в `docs/BUILD-RISKS.md`.

## Повторная сборка

Та же команда. Скрипт идемпотентен, запущенное приложение гасит сам.

Отладочная сборка (быстрее компилируется, медленнее работает):

```bash
CONFIG=debug ./scripts/build.sh
```

## Стабильная подпись и права (один раз)

По умолчанию сборка подписывается ad-hoc, и у такой подписи нет постоянной личности: cdhash меняется на каждой пересборке, macOS считает каждую сборку новым приложением. Из-за этого права Accessibility/Микрофон не переживают пересборку, а в «Универсальном доступе» копятся записи от старых сборок — удаление старой версии их не убирает (macOS не чистит права при удалении приложения, это поведение системы).

Лечится один раз — постоянным самоподписанным сертификатом:

```bash
bash scripts/setup-signing.sh
```

После этого `build.sh` подписывает им все сборки, macOS видит их как одно приложение: права выдаёшь один раз, они переживают обновления, дубликаты больше не копятся. При первой подписи macOS может один раз спросить пароль — нажми «Всегда разрешать».

Если нужна именно чистая выдача прав с нуля (проверить онбординг, раздать бету):

```bash
bash scripts/reset-permissions.sh          # сбросить все права
bash scripts/reset-permissions.sh --purge   # + удалить .app
```

Оставшиеся серые записи от прежних ad-hoc-сборок убери один раз кнопкой «−» — со стабильной подписью заново они не появятся.

## Если однажды появится Xcode

В репозитории лежит `project.yml` — спецификация XcodeGen. Тогда можно будет работать в Xcode как в IDE:

```bash
brew install xcodegen && xcodegen generate && open QuantVoice.xcodeproj
```

Основной путь сборки при этом остаётся прежним — SwiftPM.
